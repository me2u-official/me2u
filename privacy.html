Privacy Policy

Last updated: June 28, 2026

Short version: me2u never stores your files. They go directly from the sender to the receiver — we can't see them, we can't access them. No accounts, no tracking, no data collection.

1. What me2u is

me2u is a peer-to-peer (P2P) file-sharing tool that runs entirely in your browser. It uses WebRTC technology to transfer files directly between devices — no intermediary server stores, processes, or even sees your file contents.

2. What data we process

Because me2u has no accounts, no databases, and no file storage, the data we handle is minimal:

  • Signaling server (PeerJS): When you create or join a session, your browser connects to our signaling server at me2u-signal.onrender.com. This server coordinates the initial handshake between peers so they can find each other. It receives your IP address and a randomly generated session ID. No file data, file names, or file sizes ever pass through the signaling server.
  • STUN/TURN servers: WebRTC uses public STUN servers (Google) and TURN relays (openrelay.metered.ca) to establish the P2P connection. These servers see your IP address during the connection process. If a direct P2P connection can't be established, data may be relayed through the TURN server — the relay sees encrypted traffic but cannot decrypt or read its contents (WebRTC uses DTLS encryption).
  • QR code generation: We use the api.qrserver.com service to generate QR codes. Your share URL is sent to this service to create the QR image. This URL contains only the session ID — no file information.

3. What we do NOT collect

  • ❌ No accounts or registration
  • ❌ No cookies or tracking scripts
  • ❌ No analytics or telemetry
  • ❌ No file data (names, sizes, contents, or metadata)
  • ❌ No IP logging (beyond what the signaling server requires for the connection)
  • ❌ No email addresses or personal identifiers

4. How file transfers work (for your privacy)

  1. Your files stay on your device. They are never uploaded to any server.
  2. Encrypted in transit. WebRTC uses mandatory DTLS (Datagram Transport Layer Security) encryption — the same technology that secures HTTPS. Data is encrypted before it leaves your browser and decrypted only by the recipient's browser.
  3. Zero-knowledge. The signaling server doesn't know what files are being transferred, their sizes, or even that a transfer is happening. It only brokers the connection.
  4. No intermediate storage. File data flows directly from sender to receiver. It is never written to any disk on our side.
  5. Session IDs are random. The session code is a cryptographically random 16-character hex string generated by your browser. It cannot be used to identify you or your files.

5. Third-party services

Service Purpose Data shared
Render (signaling server) PeerJS hosting IP address, session ID
Google STUN NAT traversal IP address
Metered.ca TURN P2P relay fallback IP address, encrypted relay traffic

6. Data retention

We retain no data. Signaling server logs are ephemeral and used only for operational purposes. Session IDs expire when the connection closes. No transfer history, file metadata, or user records are stored.

7. Your rights

Since we collect no personal data, there is nothing to access, modify, or delete. If you have questions about this policy, open an issue on our GitHub repository.

8. Changes to this policy

If this policy changes, the "Last updated" date at the top will be revised. We encourage you to review it periodically.
