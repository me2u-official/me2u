How It Works

me2u sends files directly between browsers — no server in between. Here's exactly how.

📤 For Senders

1

📁Select your files

Click the drop zone or drag and drop files. You can select multiple files at once — there's no size limit. Your files stay on your device; nothing is uploaded yet.

2

🔗Generate a link

Click "Generate Link." Your browser creates a secure session with a random code and connects to the signaling server. A shareable link and QR code appear — send these to your recipient.

3

📤File transfers automatically

Once the recipient opens your link and accepts the file, the transfer starts. Files are read in 64 KB chunks and sent directly to the receiver. You'll see real-time progress and speed.

4

✅Done

When all files are sent, you'll see a success message. The session ends — no data is stored anywhere.

📥 For Receivers

1

🔗Open the link

Click the link the sender shared with you (or scan the QR code). The page will switch to receive mode and fill in the code automatically.

2

📥Click Connect

The app connects to the sender via the signaling server. Once connected, you'll see the file details (name, size, type).

3

✅Accept & download

Click "Accept." On desktop Chrome/Edge, you'll be asked where to save the file (streams directly to disk). On other browsers, the file downloads automatically after transfer. Keep the page open until done.

4

🎉Done

Your file is saved. If the sender sent multiple files, the next one starts automatically.

🔒 Security

me2u is designed so that we cannot access your files even if we wanted to:

  • End-to-end encrypted. WebRTC mandates DTLS encryption — the same technology that secures HTTPS traffic. Data is encrypted before it leaves the sender's browser and decrypted only by the receiver's browser.
  • Zero server storage. The signaling server only helps peers find each other. It never sees, stores, or processes file data, file names, or file sizes.
  • No accounts. No registration, no email, no passwords. Your session is identified by a random code that your browser generates.
  • Session codes are random. Generated using crypto.getRandomValues — cryptographically secure, not guessable.
  • No logs. The signaling server is configured to retain no logs. Once the connection ends, the session data is gone.

❓ Frequently Asked Questions

Is there a file size limit?

No. The code has no limit. For files over 20 GB, we recommend the recipient use desktop Chrome, Edge, or Opera — these browsers can stream directly to disk, avoiding RAM crashes.

Can I send multiple files?

Yes. Select multiple files when browsing, or drag them all into the drop zone. They'll be sent one after another — the receiver accepts each one.

How fast is the transfer?

As fast as both internet connections allow. Speed is limited by the slower of the sender's upload and the receiver's download. Check your speed at speedtest.net.

What happens if the connection drops?

The receiver can click "Reconnect & Resume" to rejoin the session. The transfer picks up from where it stopped (the sender doesn't re-send already-delivered chunks).

Is my data private?

Yes. Files are encrypted end-to-end using DTLS and never touch any server. The signaling server only handles the initial handshake. See the Privacy Policy for details.

Does it work on mobile?

Yes — but files over 150 MB may crash mobile browsers due to RAM limits. For large files, use desktop Chrome, Edge, or Opera with disk streaming support.

Do I need to install anything?

No. Everything runs in the browser. You can also install me2u as a PWA (Progressive Web App) for an app-like experience.

Can I use it behind a firewall or VPN?

Usually yes. WebRTC uses multiple techniques (STUN, TURN) to work around NATs and firewalls. If a direct connection fails, it falls back to a relay server — slower but works everywhere.
